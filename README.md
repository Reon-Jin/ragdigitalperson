# Self-RAG 数字人助手

这是当前项目的升级版，重点不再是“固定检索几个片段”，而是把问答流程做成了更接近论文思路的自适应 RAG：

- 自适应检索：先判断当前问题要不要检索，再决定浅检索还是深检索。
- 查询改写：会自动生成多个检索查询，减少用户原始提问和资料内容之间的语义偏差。
- 分层检索：先找相关书/文档，再找章节，再找具体分段，而不是直接在所有小段里盲搜。
- 可浏览资料：上传整本书后，可以在前端按“书 -> 章节 -> 分段”查看每个分段的完整内容。
- 流式回答：回答以流式方式展示。
- 实时语音输入：浏览器支持时，可直接语音输入，并在停顿后自动发送。
- 数字人情绪联动：数字人会根据回答阶段和语气切换待机、倾听、思考、说话、严肃、担忧、开心等状态。

## 技术结构

### 后端

- `FastAPI`
- `DocumentStore`
  - 解析 `txt / md / pdf / docx`
  - 自动切分章节和分段
  - 建立文档级、章节级、分段级索引
- `RagEngine`
  - 查询规划
  - 自适应检索决策
  - 查询改写
  - 分层检索
  - 流式生成
- `DeepSeekClient`
  - 普通对话
  - JSON 输出
  - 流式输出

### 前端

- 单页 Web 界面
- 书库浏览器
- 分段浏览器
- 流式聊天
- 浏览器语音识别
- 浏览器语音播报
- SVG 数字人形象与表情状态机

## 使用方式

1. 上传 PDF、DOCX、TXT 或 Markdown 资料。
2. 左侧书库会显示每本资料的摘要、章节数和分段数。
3. 点击书目后，可以进一步点开章节并查看每个分段的完整内容。
4. 在聊天框输入问题，或点击“开始语音”直接说话。
5. 系统会先规划是否检索，再分层检索，再流式返回答案。

## 当前实现说明

- 这版已经实现了论文启发式的 `Self-RAG + hierarchical retrieval + query rewriting` 工作流。
- 目前的“是否检索 / 查询改写 / 检索深度”是通过提示式规划器实现的，不是重新训练出的专用 Self-RAG 模型。
- 语音输入使用浏览器原生 Web Speech API，因此不同浏览器的支持度会有差异。
- 数字人目前是更拟人的 SVG 形象，不是真人视频驱动；如果你后续要升级到 Live2D、视频数字人或口型同步，我可以继续往下做。
